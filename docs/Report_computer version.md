# CHƯƠNG 3. NỘI DUNG CÁC LAB

## 3.1 LAB 1: Cơ bản về OpenCV và đồ họa

### 3.1.1 Mục tiêu

Làm quen với thư viện OpenCV, các thao tác cơ bản như đọc, hiển thị, lưu ảnh và hiểu về hệ tọa độ cũng như các hệ màu (BGR, RGB, Grayscale).

### 3.1.2 Lý thuyết liên quan

- **Hệ tọa độ ảnh:** Gốc (0,0) ở góc trên bên trái, trục x hướng sang phải, trục y hướng xuống dưới.
- **Hệ màu:** OpenCV mặc định sử dụng BGR, trong khi Matplotlib sử dụng RGB. Cần chuyển đổi bằng `cv2.cvtColor`.

### 3.1.3 Dữ liệu đầu vào

Ảnh gốc `data/input/in.jpg` kích thước 3840 x 2400 pixels.

### 3.1.4 Thuật toán / Quy trình xử lý

1. Đọc ảnh bằng `cv2.imread()`.
2. Kiểm tra thông số ảnh (shape, dtype).
3. Chuyển đổi từ BGR sang RGB và hiển thị bằng Matplotlib.
4. Lưu ảnh đã xử lý bằng `cv2.imwrite()`.

### 3..1.5 Code chính

```python
import cv2
import matplotlib.pyplot as plt

original_bgr_image = cv2.imread('../data/input/in.jpg')
original_rgb_image = cv2.cvtColor(original_bgr_image, cv2.COLOR_BGR2RGB)

plt.imshow(original_rgb_image)
plt.title("Ảnh gốc (Hệ màu RGB)")
plt.axis('off')
plt.show()

height, width, channels = original_bgr_image.shape
print(f"Kích thước ảnh: {width} x {height} pixels, {channels} kênh màu")
```

### 3.1.6 Kết quả thực nghiệm

- Ảnh hiển thị đúng màu sắc.
- Thông số: 3840 x 2400 pixels, 3 kênh màu (uint8).

### 3.1.7 Nhận xét

Thư viện OpenCV dễ sử dụng. Việc nắm vững sự khác biệt giữa BGR và RGB là cực kỳ quan trọng.

### 3.1.8 Khó khăn và cách khắc phục

- **Khó khăn:** Ảnh hiển thị sai màu bằng Matplotlib.
- **Khắc phục:** Sử dụng `cv2.cvtColor` để chuyển sang RGB.

## 3.2 LAB 2: Xử lý ảnh cơ bản - Toán tử điểm ảnh và Lọc tuyến tính

### 3.2.1 Mục tiêu

Thực hiện các phép biến đổi pixel (độ sáng, tương phản, âm bản, ngưỡng hóa) và các bộ lọc tuyến tính (Mean, Gaussian) để xử lý ảnh.

### 3.2.2 Lý thuyết liên quan

- **Toán tử điểm ảnh:** Tác động trực tiếp lên từng pixel độc lập (ví dụ: $I_{out} = I_{in} + c$).
- **Lọc tuyến tính:** Sử dụng tích chập với Kernel để tính giá trị pixel dựa trên các pixel lân cận.
- **Lọc Gaussian:** Trọng số pixel trung tâm cao nhất, giảm dần ra xa, giúp làm mịn ảnh tự nhiên.

### 3.2.3 Dữ liệu đầu vào

Ảnh `data/input/test.jpg`.

### 3.2.4 Thuật toán / Quy trình xử lý

1. Chuyển ảnh sang Grayscale.
2. Thay đổi độ sáng/tương phản dùng `np.clip`.
3. Biến đổi âm bản và cắt ngưỡng (Thresholding).
4. Áp dụng bộ lọc Mean và Gaussian với các kích thước Kernel khác nhau.

### 3.2.5 Code chính

```python
# Tăng độ sáng
img_brighter = np.clip(img_gray.astype(np.int32) + 100, 0, 255).astype(np.uint8)
# Cắt ngưỡng
_, img_thresh = cv2.threshold(img_gray, 128, 255, cv2.THRESH_BINARY)
# Lọc Gaussian
img_blurred = cv2.GaussianBlur(img_gray, (5, 5), sigmaX=1.0)
```

### 3.2.6 Kết quả thực nghiệm

- Ảnh sáng hơn rõ rệt khi cộng hằng số.
- Lọc Gaussian 5x5 giúp loại bỏ nhiễu mịn mà vẫn giữ được cấu trúc chính của ảnh.

### 3.2.7 Nhận xét

Các phép toán điểm ảnh đơn giản nhưng hiệu quả trong việc tiền xử lý. Lọc Gaussian thường cho kết quả tốt hơn lọc Mean trong việc bảo toàn chi tiết.

### 3.2.8 Khó khăn và cách khắc phục

- **Khó khăn:** Hiện tượng tràn số khi cộng giá trị pixel.
- **Khắc phục:** Chuyển sang kiểu dữ liệu `int32` trước khi cộng và dùng `np.clip` để giới hạn trong khoảng [0, 255].

## 3.3 LAB 3: CANNY EDGE DETECTOR

### 3.3.1 Mục tiêu

Hiểu và triển khai thuật toán phát hiện cạnh Canny – một trong những thuật toán phát hiện cạnh tối ưu nhất hiện nay.

### 3.3.2 Lý thuyết liên quan

Thuật toán gồm 5 bước:

1. **Giảm nhiễu:** Dùng bộ lọc Gaussian.
2. **Tính Gradient:** Dùng toán tử Sobel để tìm độ lớn và hướng của cạnh.
3. **Non-maximum Suppression:** Làm mỏng các cạnh.
4. **Double Thresholding:** Phân loại cạnh mạnh, yếu và không phải cạnh.
5. **Hysteresis Tracking:** Kết nối các cạnh yếu vào cạnh mạnh.

### 3.3.3 Dữ liệu đầu vào

Ảnh `data/input/test.jpg`.

### 3.3.4 Thuật toán / Quy trình xử lý

1. Chuyển ảnh sang Grayscale.
2. Làm mịn ảnh bằng `cv2.GaussianBlur`.
3. Áp dụng `cv2.Canny` với các bộ thông số (threshold1, threshold2) khác nhau.
4. Quan sát sự thay đổi của các cạnh dựa trên ngưỡng.

### 3.3.5 Code chính

```python
# Làm mịn
img_blurred = cv2.GaussianBlur(img_gray, (5, 5), sigmaX=1.0)
# Áp dụng Canny
canny_edges = cv2.Canny(img_blurred, threshold1=50, threshold2=150)
```

### 3.3.6 Kết quả thực nghiệm

- Với ngưỡng (50, 150), các cạnh chính của vật thể được giữ lại rất tốt, ít bị nhiễu.

## 3.4 LAB 4: So sánh sự tương đồng của các hình ảnh sử dụng Wavelet, Python

### 3.4.1 Mục tiêu

Xây dựng hệ thống so sánh độ tương đồng giữa các hình ảnh bằng cách sử dụng biến đổi Wavelet (DWT) để trích xuất đặc trưng và tính toán khoảng cách Hamming.

### 3.4.2 Lý thuyết liên quan

- **Discrete Wavelet Transform (DWT):** Phân tích ảnh thành các băng tần khác nhau. Băng tần LL (Low-Low) chứa thông tin tóm tắt cốt lõi của ảnh.
- **Wavelet Hash (wHash):** Tạo mã hash từ băng tần LL để đại diện cho ảnh.
- **Hamming Distance:** Số lượng vị trí bit khác nhau giữa hai mã hash. Khoảng cách càng nhỏ, ảnh càng tương đồng.

### 3.4.3 Dữ liệu đầu vào

- Bộ dữ liệu `data/DWT/images`.
- File CSV `data/DWT/pairs.csv` chứa thông tin các cặp ảnh và nhãn (giống/khác).

### 3.4.4 Thuật toán / Quy trình xử lý

1. Đọc và tiền xử lý ảnh (chuyển sang Grayscale, resize).
2. Áp dụng DWT (sử dụng wavelet 'haar').
3. Trích xuất thành phần LL và tạo mã bit (hash).
4. Tính toán khoảng cách Hamming và độ tương đồng.
5. Đánh giá bằng Accuracy, F1-score và đường cong ROC.

### 3.4.5 Code chính

```python
import pywt
# Biến đổi DWT 2D
coeffs = pywt.dwt2(pixels, 'haar')
LL, _ = coeffs
# Tạo hash bit dựa trên giá trị trung bình
hash_bit = (LL > LL.mean()).flatten()
# Tính khoảng cách Hamming
dist = int(np.sum(hash1 != hash2))
```

### 3.4.6 Kết quả thực nghiệm

- Hệ thống đạt độ chính xác (Accuracy) trên 80% đối với tập dữ liệu Test.
- Ngưỡng tối ưu thường nằm trong khoảng 0.7 - 0.75 để đạt F1-score cao nhất.

### 3.4.7 Nhận xét

Wavelet Hash rất hiệu quả trong việc tìm kiếm ảnh tương đồng (near-duplicate) vì nó bỏ qua các biến đổi nhỏ về ánh sáng và nhiễu.

### 3.4.8 Khó khăn và cách khắc phục

- **Khó khăn:** Một số cặp ảnh có nội dung giống nhau nhưng góc chụp khác nhiều khiến mã hash thay đổi.
- **Khắc phục:** Kết hợp thêm các phương pháp Feature Matching truyền thống (như ORB/SIFT) để tăng độ bền vững.

## 3.5 LAB 5: Nhận diện khuôn mặt thời gian thực với FaceNet & MTCNN trên Webcam

### 3.5.1 Mục tiêu

Triển khai hệ thống nhận diện khuôn mặt thực tế qua Webcam, có khả năng đăng ký người mới và ghi nhật ký điểm danh tự động.

### 3.5.2 Lý thuyết liên quan

- **MTCNN (Multi-task Cascaded Convolutional Networks):** Phát hiện vị trí khuôn mặt và 5 điểm mốc (mắt, mũi, miệng) cực kỳ chính xác.
- **FaceNet:** Chuyển đổi ảnh khuôn mặt thành vector 512 chiều (Embedding). Các vector của cùng một người sẽ ở gần nhau trong không gian vector.
- **Cosine Similarity:** Tính độ tương đồng giữa vector khuôn mặt hiện tại và các vector trong cơ sở dữ liệu.

### 3.5.3 Dữ liệu đầu vào

- Luồng video trực tiếp từ Webcam.
- Cơ sở dữ liệu khuôn mặt đã đăng ký tại `data/registered_faces`.

### 3.5.4 Thuật toán / Quy trình xử lý

1. Chụp khung hình từ Webcam.
2. Dùng MTCNN để tìm và cắt khuôn mặt.
3. Dùng FaceNet để trích xuất Embedding 512 chiều.
4. So sánh với database dùng Cosine Similarity.
5. Hiển thị tên (hoặc "Unknown") và ghi log vào file CSV nếu nhận diện thành công.

### 3.5.5 Code chính

```python
# Phát hiện khuôn mặt
results = detector.detect_faces(image_rgb)
# Trích xuất đặc trưng
embedding = embedder.embeddings(np.expand_dims(face_pixels, axis=0))[0]
# So sánh
score = cosine_similarity(embedding.reshape(1, -1), ref_emb.reshape(1, -1))[0][0]
```
